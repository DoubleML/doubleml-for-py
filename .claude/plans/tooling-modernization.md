# Contributing Guide & Tooling Modernization

## Context
Audit of `CONTRIBUTING.md` against `pyproject.toml`, `.pre-commit-config.yaml`, and the
GitHub workflow surfaced bugs and drift. The user wants to modernize the toolchain by
adopting `uv` (lockfile + fast installer) and replacing `black` with `ruff format`, while
keeping the rollout safe alongside the in-flight `sk-refactoring` branch.

`.pytest_cache/README.md` is auto-generated by pytest and out of scope.

### Decisions taken
- **Mypy**: keep config + dev dep, no CI gate yet (codebase not yet clean; gradual adherence).
- **Tool target**: ruff `target-version = "py310"`, mypy `python_version = "3.10"` (track minimum supported Python).
- **Black `preview`**: remove.
- **uv**: full adoption with `uv.lock`, CI via `uv sync`, document `uv run`.
- **Formatter**: switch black → ruff format, staged so reformat noise is isolated.
- **PR shape**: two PRs (low-risk first, tooling second).
- **Persistence**: track work in a GitHub issue; commits below become its checklist.
- **Timing**: start after `sk-refactoring` lands on `main` to avoid format-churn conflicts.

## Findings (the "why")

### Bugs
- `CONTRIBUTING.md:37` prints `sklearn.__version__` for the DoubleML version line and references `sklearn` before it is imported.
- `CONTRIBUTING.md:85` install command `pip install --editable .[dev, rdd]` has a stray space inside extras (breaks under zsh) and disagrees with CI's `pip install -e .[dev,rdd]` (`pytest.yml:49`).

### Configuration smells
- `pyproject.toml:29` declares `mypy>=1.18.0` as a **runtime** dependency. End users get mypy when they `pip install doubleml`.
- `pyproject.toml:112` ruff `target-version = "py312"` while project supports `>=3.10`.
- `pyproject.toml:127` mypy `python_version = "3.12"` for the same reason.
- `pyproject.toml:96` Black `preview = true` causes formatting drift across Black versions even at a pinned rev.

### Documentation drift
- Pre-commit is the actual CI gate (`pytest.yml:51-54`) but `CONTRIBUTING.md:133-147` labels it "(Optional)".
- `ci` and `ci_rdd` pytest markers (`pyproject.toml:66-69`) are undocumented for contributors.
- Doctests run by default (`pyproject.toml:83-86` adds `--doctest-modules`) — undocumented.
- Mypy: codebase not yet mypy-clean — needs documentation, not a CI gate.

## Risks & mitigations
- **Format-churn collisions with `sk-refactoring`** → defer PR 2 until that branch merges. PR 1 has no reformatting and is safe in parallel.
- **`uv.lock` merge conflicts** → document in CONTRIBUTING that `uv lock` must be re-run when dependencies change; treat lockfile conflicts the same as `package-lock.json` conflicts (regenerate, don't hand-merge).
- **`ruff format` ≠ `black` edge cases** → commit 2 of PR 2 surfaces them as a discrete diff before the formatter switch, so any disagreements are visible and reviewable.
